# 《When Compression Becomes an Attack Surface: Black-Box Attacks on Prompt-Compressed LLM Agents》
> 论文来源：arXiv:2510.22963v4 [cs.CR](https://arxiv.org/pdf/2510.22963)，香港科技大学团队，代码开源：https://github.com/zsLiu2003/Comattack

## 一、核心研究背景与问题提出
### 1.1 提示压缩的普及与安全边界偏移
LLM Agent 是由系统提示、工具规范、检索文档、多轮交互历史等多组件拼接而成的流水线，随着上下文长度增长，提示词带来了**延迟升高、成本增加、关键信息被淹没**三大问题。为此，提示压缩（Prompt Compression）模块被广泛集成到 LangChain、LlamaIndex、PromptFlow 等主流 Agent 框架中，通过抽取或改写的方式压缩提示长度，满足 Token 预算约束。

但论文指出一个被忽视的安全本质：**压缩器处于提示组装与LLM推理之间的安全关键路径上**。启用压缩后，后端LLM不再基于原始提示推理，而是基于压缩器输出的有损压缩结果进行推理。压缩器直接决定了哪些指令、证据、上下文能够保留并影响下游行为，这使得系统的有效安全边界从后端LLM本身，偏移到了「压缩+推理」的整条流水线。

### 1.2 新型攻击面：共享预算下的可信/不可信输入共压缩
当可信输入（如系统提示、安全护栏）与不可信输入（如用户查询、外部检索内容）在**同一个压缩预算下共同压缩**时，攻击者仅需扰动不可信输入部分，就能影响可信内容的保留结果。
- 典型示例：在违规查询后追加一段无意义的对抗后缀，会导致压缩器丢弃系统提示中「必须永远禁止」这类关键安全否定词，最终后端LLM收到残缺的压缩提示，执行原本被禁止的操作。
- 与传统攻击的本质区别：提示注入、越狱、RAG投毒的目标都是后端LLM，要求恶意内容必须存活到推理阶段并被模型解读；而本攻击的目标是**压缩器本身**，扰动不需要包含任何有意义的指令，甚至不需要在压缩后存活，其唯一作用是改变压缩器的保留决策，让关键信息在LLM推理前就被丢弃。

## 二、核心理论：对抗性信息丢失（AIL）
论文将上述漏洞形式化为**对抗性信息丢失（Adversarial Information Loss, AIL）**，定义为：攻击者通过操控有损压缩器，超出良性压缩本身所带来的额外下游行为扭曲。

### 2.1 基础定义
- **提示压缩模型**：带预算的变换函数 $C_R$，将原始提示 $X$ 映射为更短的压缩提示 $\bar{X}$，满足资源约束 $r(\bar{X}) \le R$（$r(\cdot)$ 衡量Token数/推理成本，$R$ 为预算）。分为两类：
  - 抽取式压缩器：直接从原文中选择保留Token/片段，措辞不变，如 LLMLingua、Selective Context；
  - 抽象式压缩器：通过改写/摘要生成更短的表述，如基于小LLM的总结压缩。
- **压缩不可区分类**：$[x]_R \triangleq \{x' \in X: C_R(x')=C_R(x)\}$，即所有经过压缩后得到相同结果的原始提示集合。压缩本质是多对一的映射，必然存在碰撞。

### 2.2 良性信息丢失（BIL）
良性输入本身经过压缩后，也会产生一定的行为偏差，称为**良性信息丢失（Benign Information Loss, BIL）**：
$$BIL(x;R) := d_M(x, C_R(x))$$
其中 $d_M$ 衡量原始提示与压缩提示在下游LLM输出分布上的差异（如KL散度、任务损失）。BIL是有损压缩的固有代价，不属于安全漏洞。

### 2.3 对抗性信息丢失（AIL）
攻击者在可行扰动集合 $N(x)$ 内优化预压缩提示，使压缩结果发生变化，带来的最大下游扭曲为最坏情况扭曲（WCD）：
$$WCD(x;R) := \sup_{x' \in N(x)} d_M(x, C_R(x'))$$

**AIL 定义为最坏情况扭曲减去良性信息丢失**，即攻击者额外造成的行为偏差：
$$AIL(x;R) := WCD(x;R) - BIL(x;R)$$

- 核心性质：$AIL \ge 0$；当且仅当所有可行扰动都属于同一个压缩不可区分类时，$AIL=0$，此时压缩器对扰动鲁棒。
- 防御启示：减少AIL的两种思路——让压缩器在扰动范围内更稳定（扩大不可区分类，但可能增大BIL），或拒绝压缩结果不稳定的输入（可能产生误报），因此存在效用-安全的权衡。

## 三、威胁模型
### 3.1 攻击目标与向量
- 直接目标：提示压缩器；最终目标：操控LLM Agent的下游行为。
- 攻击原理：利用压缩预算瓶颈，通过在不可信输入中加入对抗扰动，诱导压缩器为满足预算而丢弃关键指令/证据。
- 提示组成：$x = x^{sys} \oplus x^{ctx} \oplus x^{que}$
  - $x^{sys}$：可信输入（系统提示），攻击者无法直接访问；
  - $x^{ctx}$：外部内容（不可信）；
  - $x^{que}$：用户查询（不可信）。

### 3.2 攻击者能力（迁移式黑盒模型）
- **知识与访问**：仅能提交输入、观察Agent最终输出，无法访问模型参数、真实压缩结果、系统提示；仅知道部署了提示压缩，可自行构建代理模型。
- **扰动向量**：
  1. **查询扰动**：攻击者以用户身份，在查询后追加有限长度的对抗后缀；
  2. **外部内容扰动**：通过上传恶意文档、投毒检索网页等方式，修改Agent获取的外部上下文。

### 3.3 攻击目标
1. **护栏削弱（Guardrail Attenuation）**：让系统提示中的安全护栏在压缩中被丢弃，突破安全限制；
2. **证据扭曲（Evidence Distortion）**：控制任务关键证据的留存，误导后端LLM的决策（如工具选择、问答结果）。

## 四、COMA 攻击方法
论文提出 **COMA（COmpression-Manipulation Attack）**，一种基于迁移的黑盒攻击方法，通过攻击者侧的代理压缩器与代理LLM，优化预压缩扰动来利用AIL漏洞。

### 4.1 问题建模：双层优化
攻击本质是一个双层优化问题：
- 外层：攻击者优化预压缩输入 $x_{atk}$，最大化AIL；
- 内层：压缩器在预算约束下，最小化自身的保真度损失，输出压缩结果。

直接求解存在三大困难：无法观察压缩结果、无法单独调用压缩器、AIL是分布级指标难以计算。因此论文采用**代理目标+两阶段近似**的策略求解。

### 4.2 近似策略
1. **代理目标替换**：用LLM评判器对输出文本打分，作为可测量的代理指标 $\hat{d}(x_{atk})$，替代分布级的AIL；
2. **代理模型替代**：用攻击者本地的代理后端LLM $\hat{M}$ 和代理压缩器 $\hat{C}_\rho$，模拟目标系统；
3. **两阶段解耦**：先在压缩空间中找到能诱导错误行为的目标提示，再搜索能让代理压缩器输出该目标的预压缩扰动。

### 4.3 阶段一：丢弃目标选择
目标：找到一个压缩后的目标提示 $\tilde{x}_{tgt}$，该提示删除了少量关键内容，但足以让后端LLM产生错误行为。
- 流程：
  1. 从正常压缩提示出发，用小模型识别关键Token/词；
  2. 贪心逐个删除关键内容，每删除一次就用代理LLM+评判器验证行为是否翻转；
  3. 首次出现行为翻转时停止，得到最小改动的目标压缩提示。
- 优势：保证目标改动量小，更容易在第二阶段通过扰动实现；同时确保目标确实能诱导错误行为。

### 4.4 阶段二：对抗扰动搜索
目标：找到约束内的扰动 $\delta$，使得原始提示加扰动后，经过代理压缩器的输出匹配阶段一的目标 $\tilde{x}_{tgt}$。
- 优化目标：最小化压缩空间匹配损失 $\mathcal{L}_{match}$，即压缩器生成目标Token的对数概率之和的负数；
- 优化方法：借鉴GCG的贪心Token替换策略，计算可编辑Token嵌入的梯度，逐轮选择最能降低损失的替换；
- 通用后缀优化：针对系统提示不可见的场景，在多个候选系统提示的集合上优化通用对抗后缀，提升跨系统的迁移性。

### 4.5 端到端验证
所有候选扰动最终在真实黑盒目标系统上验证，用同样的LLM评判器判定攻击是否成功，选择改动最小的成功扰动作为最终结果。

## 五、实验设置
### 5.1 三大任务与数据集
| 任务 | 攻击类型 | 数据集规模 | 攻击成功定义（ASR） |
|------|----------|------------|---------------------|
| Agent工具选择（ATS） | 证据扭曲 | 249条，含172类产品、77类工具 | 干净输入选对工具、攻击后选错的比例 |
| 问答（Q&A） | 证据扭曲 | SQuAD筛选后913条 | 攻击后答案与标准答案不匹配的比例 |
| 系统提示破坏（SPC） | 护栏削弱 | 筛选后1563条（来自泄露系统提示+RealGuardrails） | 攻击后模型违反安全护栏的比例 |

所有数据集都经过筛选：排除干净压缩就出错的样本，也排除删掉关键内容后仍不犯错的样本，确保ASR真实反映攻击带来的效果。

### 5.2 测试压缩器与模型
- **抽取式压缩器（3个）**：Selective Context (SC)、LLMLingua、LLMLingua2
- **抽象式压缩器（3个）**：基于 Llama-3.2-3B、Qwen3-4B、Gemma-3-4B 的指令式摘要压缩
- 默认代理后端LLM：Llama-3.1-8B-instruct
- 评判LLM：Qwen3-235B-A22BInstruct-2507-FP8
- 扰动预算：32个Token（占2048总长度的1.56%）
- 压缩预算定义：保留Token的比例，优化时候选预算为{0.5, 0.6, 0.7}

### 5.3 基线方法
均为非压缩感知的传统攻击，同样施加在预压缩输入上，再经过压缩流水线：
1. Naive Attack：随机字符级扰动
2. Direct Injection：追加显式恶意指令
3. Escape Characters：插入转义分隔符
4. Context Ignore：插入「忽略之前指令」等覆盖短语
5. JudgeDeceiver：基于梯度的对抗提示生成（迁移黑盒设置）

## 六、实验结果与分析
### 6.1 RQ1：攻击有效性
- **核心结果**：COMA在全部18个测试场景（6个压缩器×3个任务）中均取得最高ASR，**平均ASR达0.71**，而最强基线仅0.21，绝对提升0.50（约3.4倍）。
- **漏洞归因验证**：
  - 无攻击的干净压缩平均ASR仅0.01，说明压缩本身几乎不导致错误；
  - 无压缩时COMA的ASR也接近0（0.01），说明COMA在没有压缩的系统上无效，其攻击性完全来自对压缩漏洞的利用。
- **任务差异**：
  - ATS任务最脆弱（平均0.92），因为直接修改被压缩的外部内容，容易操控证据留存；
  - SPC任务难度更高（平均0.54），因为依赖后缀向未知系统提示的迁移。
- **压缩类型差异**：抽取式压缩器平均ASR 0.76，高于抽象式的0.67；抽取式的保留决策更规律，更容易被操控。

### 6.2 RQ2：泛化能力
#### 跨压缩预算泛化
在预算R∈{0.2, 0.4, 0.6, 0.8}下测试，COMA平均ASR达0.69，泛化性良好：
- 预算极低（R=0.2）时，良性压缩本身就会造成大量错误（无攻击ASR 0.60）， adversarial放大空间有限（攻击-良性差值仅0.14）；
- **攻击放大效应在部署常用预算（0.6~0.8）下达到峰值**：此时良性系统准确率很高（无攻击ASR≈0），但COMA仍能稳定诱导错误（ASR 0.69/0.65），差值达0.68~0.65，安全风险最突出。

#### 跨后端LLM泛化
在6种不同家族、不同规模的后端LLM（Llama-2-7B、Llama-3-8B、Llama-3-70B、Mistral-2-14B、Qwen3-7B、Qwen3-32B）上测试：
- 整体平均ASR 0.69，范围0.60~0.77，一致性强；
- 机制解释：攻击发生在压缩阶段，关键信息被删除后，任何后端都无法恢复；
- 任务差异：ATS跨模型最稳定（0.90~0.92），Q&A和SPC受后端能力影响更大——强模型可靠参数知识补全问答、靠对齐能力坚守护栏。

### 6.3 RQ3：消融实验与攻击机制验证
#### 代理模型迁移性
测试代理压缩器与目标压缩器不匹配时的攻击效果：
- 12组不匹配场景的平均ASR仍达0.65，仅比最佳匹配低0.08；
- 抽取式中，Llama-2-7B对SC/LLMLingua效果最好，XLM-R-large对LLMLingua2效果最好；
- 抽象式跨模型迁移衰减很小（Δ=-0.01~-0.09）；
- 结论：不需要精确复现目标压缩器，只要代理能近似「什么信息会被保留」，攻击就能迁移。

#### 关键令牌移除率（CTRR）
衡量COMA是否真的删除了阶段一识别的关键Token：
- 整体CTRR 0.76，与ASR 0.72高度对齐；
- ATS任务CTRR最高（0.96），与ASR几乎一致；Q&A存在差距，因为答案证据更分散、有冗余；SPC完全匹配（0.54/0.54）；
- 结论：**COMA的成功机制是可控地删除行为关键信息**，而非靠注入通用恶意内容。

### 6.4 RQ4：评判器可靠性
在300个分层抽样样本上对比LLM评判器与人工标注：
- 一致性98.7%，Cohen's κ=0.95；
- 攻击失败类的F1值95.5%；
- 验证了LLM评判器是可靠的攻击成功判定标准。

### 6.5 RQ5：真实系统案例
将优化的扰动迁移到两个真实Agent流水线：
1. **VSCode Cline**：原本会拒绝的请求，攻击后触发了对敏感文件的特权读取操作；
2. **LangChain + Ollama**：特征提取查询被攻击后，从正确模型重定向到了BERTOverflow。
- 结论：COMA攻击可迁移到真实部署的Agent系统，切实改变安全执行与工具调用行为。

### 6.6 RQ6：防御评估
#### 现有防御的失效
测试了两类主流防御，效果均大幅下降：
- 检测类：困惑度(PPL)、窗口困惑度(Window-PPL)，平均DSR仅0.04、0.07；
- 预防类：StruQ、SecAlign等鲁棒后端，平均DSR均为0.04；
- 原因：传统防御针对的是「恶意内容本身」，而COMA的扰动不需要存活、不需要有语义，压缩步骤会消解检测特征。

#### 本文提出的防御：隔离压缩（IC）
核心思路：将可信输入与不可信输入分开压缩，各自使用独立预算，再用显式分隔符拼接后送入后端LLM，并标记不可信内容。
- 效果：SPC任务平均DSR达0.96，效果极其显著——从根源上避免了不可信内容与可信护栏争夺压缩预算；
- ATS与Q&A任务提升较小，但仍高于现有防御，因为标记不可信内容能激活后端的交叉校验能力。

## 七、相关工作
1. **提示压缩**：分为离散压缩（抽取式、抽象式，可通用部署）与连续压缩（依赖特定模型架构）；SecurityLingua是安全导向的压缩防御，不属于通用压缩器。
2. **Agent漏洞与防御**：现有攻击包括提示注入、越狱、RAG投毒等，均针对后端LLM，要求恶意内容存活到推理阶段；现有防御分为检测类（困惑度等）与预防类（结构分离、对齐增强），均未考虑压缩带来的新攻击面。

## 八、核心贡献与结论
### 主要贡献
1. 识别出提示压缩器是LLM Agent的新型攻击面，证明效率组件会实质性改变系统安全边界；
2. 形式化提出「对抗性信息丢失（AIL）」概念，刻画了有损压缩被攻击者利用带来的额外行为扭曲；
3. 提出COMA迁移式黑盒攻击，在6种压缩器、3类任务上均显著优于基线，且具备强泛化性；
4. 全面验证了攻击的真实影响，并提出隔离压缩的防御方案，显著提升鲁棒性。

### 核心结论
提示压缩并非安全中立的优化组件，它会从根本上改变LLM Agent的行为与攻击面。当可信与不可信输入共享压缩预算时，攻击者可以通过极小的不可信输入扰动，诱导压缩器丢弃关键安全护栏与任务证据，进而操控Agent行为。将可信与不可信输入隔离压缩，是目前最有效的缓解手段。
